# Senior Quant Case Study  

### Overview
This weekend challenge assesses your ability to solve real-world problems combining **machine learning**, **mathematical optimization**, and **software engineering**. Choose ONE problem that excites you, build a working solution, and demonstrate it back to us in 15-minutes.

**Time Limit**: Friday 9am → Tuesday 9am (10-15 hours expected effort)

**Presentation**: 15-minute live demo + 15-minute Q&A

**Resources**: All online resources, AI assistants, and libraries are encouraged. Acknowledge your sources. We dont mind if you get AI and AI agents involved - in fact we embrace and encourage it
just dont blindly copy and paste, because if they made a mistake or did something illogical, then YOU made the mistake and did something illogical.

---

## What We're Assessing

We want to see how you approach problems requiring:
- **ML Modeling (33%)**: Feature engineering, model selection, validation
- **Mathematical Optimization (33%)**: Constraint satisfaction, objective optimization
- **Software Engineering (33%)**: Clean code, working solution, practical thinking

**Most importantly**: We value **working solutions** over perfect code, **clear thinking** over complex algorithms, and **creative problem-solving** over following templates.

## Evaluation Rubric

Total: 100 points across three dimensions

### 1. ML Modeling (25 points)
- Model selection and justification (10 pts)
- Feature engineering creativity (5 pts)
- Validation methodology (5 pts)
- Performance on test set (5 pts)

### 2. Mathematical Optimization (25 points)
- Problem formulation (10 pts)
- Algorithm choice and efficiency (5 pts)
- Constraint handling (5 pts)
- Solution quality (5 pts)

### 3. Software Engineering (25 points)
- Code quality and structure (5 pts)
- Working demo/visualization (10 pts)
- Scalability considerations (5 pts)
- Testing and error handling (5 pts)

### Presentation (25 points)
- Clear explanation (15 pts)
- Answering questions (10 pts)

## Submission Requirements

### What to Submit (by Tuesday 9 AM)
1. **Code Repository** (GitHub/GitLab or ZIP)
   - README.md with setup + how to run
   - requirements.txt (dependencies)
   - main.py (entry point)
   - src/ (your implementation)

2. **Brief Writeup** (1-2 pages, can be in README)
   - High-level approach
   - Key design decisions and tradeoffs
   - Results summary (metrics vs baseline)
   - Known limitations

3. **Runnable Demo**
   - Works on standard laptop (no GPU, no cloud)
   - Clear instructions in README

### The Presentation (Monday, 15 minutes)
- **15 minutes**: Live demo of your solution
- **15 minutes**: Q&A with technical panel

**Show:**
1. Quick overview (2 min): Problem + approach
2. Live demo (5 min): Working features
3. Results (5 min): Metrics vs baseline
4. Insights (3 min): Key findings/challenges

## Practical Guidance

### Time Allocation (10-15 hours)
- **(2-3h)**: Understand problem, setup environment
- **(4-5h)**: ML + Optimization implementation
- **(3-5h)**: Integration, visualization, benchmarking
- **(1-2h)**: Final testing, presentation prep

### What Impresses Us ✅
- Working solution > Perfect code
- Clear thinking > Complex algorithms
- Creative insights > Following templates
- Good tradeoffs > Over-engineering

### Red Flags ❌
- Copy-pasted code you can't explain
- 90% data wrangling, 10% modeling (Getting to the end is more important than perfection)
- "I would have done X but ran out of time"
